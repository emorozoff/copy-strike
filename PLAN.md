# CopyS — браузерный Counter-Strike для двоих

Клон CS (1v1 по сети) целиком в браузере. Статика на GitHub Pages, соединение игроков — WebRTC P2P, движок — Three.js. Играют два человека: хост и друг, оба предположительно в РФ (это влияет на выбор сетевой инфраструктуры).

**Конечная цель (максимальная стадия):** Dust2, режим закладки бомбы с раундами и экономикой, покупка оружия (нож/пистолеты/AK/M4/AWP), гранаты, броня, радар, звуки и анимации — «настоящий CS», но для двоих.

## 1. Ключевые технические решения

Все решения ниже основаны на проверенном исследовании (ссылки открывались и проверялись).

### Карта — готовый GLB Dust2 (проверено скачиванием)

- **Основной вариант:** `https://github.com/decentraland/cs-1.6/raw/main/assets/scene/de_dust2_-_cs_map.glb`
  Валидный GLB 1.43 МБ, ~9 300 треугольников, 34 встроенные PNG-текстуры (оригинальные текстуры CS 1.6). Это авто-конвертация модели «de_dust2 - CS map» (pancakesbassoondonut, Sketchfab, CC-BY, 38k скачиваний).
- Модель в единицах GoldSrc (1 юнит ≈ 1 дюйм). **Масштаб ×0.0254 → метры**: карта ~114×135 м, перепад высот ~15 м, пропорции аутентичные (рост игрока CS 72 юнита ≈ 1.83 м, глаза на 64 ≈ 1.63 м).
- 9.3k треугольников — рендер-меш можно использовать напрямую как коллизионный (через BVH), без упрощения.
- **Запасной вариант (красивее):** «De_dust2» от Leaf_dev на Sketchfab — версия из CS:Source, 109k треугольников, 100 текстур, CC-BY, бесплатный аккаунт. Рендерится нормально, но коллизии лучше считать по упрощённому прокси-мешу. Есть и «De_Dust2_2020» (58k) как середина.
- Дорогой путь на будущее: экспорт карты из бесплатной CS2 через Source 2 Viewer в glTF (миллионы треугольников, нужен день работы в Blender на децимацию) — только если захочется современную графику.

### Стек рендера и физики

- **Three.js** (ES-модули) + **three-mesh-bvh** — одна BVH-структура по мешу карты обслуживает всё: коллизии капсулы игрока, hitscan-лучи выстрелов, отскоки гранат, декали.
- Контроллер персонажа — паттерн из `three-mesh-bvh/example/characterMovement.js`: капсула + `shapecast` + выталкивание из треугольников. Устойчив к «грязной» геометрии импортированной карты.
- **Не использовать** Octree из официального примера three.js `games_fps` — задокументированы туннелирование сквозь стены (issue #21921), дрожание в углах и фризы `fromGraphNode` на больших картах.
- Hitscan: пропатчить `Mesh.raycast` → `acceleratedRaycast`, `raycaster.firstHitOnly = true`. Сначала луч в карту (окклюзия), потом дешёвая проверка капсулы/сфер противника.
- **PointerLockControls** + игровой цикл с фиксированным шагом (аккумулятор, 60 Гц тиков, рендер отвязан) — фиксированные тики нужны и неткоду. Дельту клампить (макс 0.1 с) от переключения вкладок.
- Оружие от первого лица: пушка на слое 1, вторая камера (FOV ~55 против ~90 у мира), `autoClear=false` + `clearDepth()` вторым проходом — ствол не втыкается в стены. Вспышка — additive-спрайт, трассеры — растянутый квад, дырки от пуль — `DecalGeometry` с кольцевым буфером ~50 штук.
- Референсы для подглядывания: `felixgren/three-arena` (формат сетевых сообщений, интерполяция чужого игрока), `mohsenheydari/three-fps` (структура entity/component, обращение с viewmodel).

### Сеть — Trystero + два DataChannel

- **Trystero** (v0.25+, активно развивается) со стратегией **Nostr** по умолчанию: сотни публичных релеев для сигналинга, работает с чисто статического сайта, без аккаунтов и ключей. Комната = `joinRoom({appId, password}, roomCode)` — идеально для двух друзей.
- **Не PeerJS**: его облачный сигналинг `0.peerjs.com` — единственная точка отказа с историей падений, а `reliable:false` в PeerJS даёт только unordered-канал (проверено по исходникам — `maxRetransmits` не выставляется вообще).
- Два канала данных на одном соединении:
  - `state` — `{ordered:false, maxRetransmits:0}` (UDP-подобный): снапшоты позиций 30 Гц, пакет ≤ ~40 байт;
  - `events` — надёжный упорядоченный: попадания, смерти, бомба, раунды, счёт, чат, кейфреймы, heartbeat.
  Trystero не даёт unreliable-канал из коробки → после коннекта взять `RTCPeerConnection` из `room.getPeers()` и создать канал руками (`negotiated:true, id` фиксированный с обеих сторон). ⚠️ API `getPeers()` исследованием не подтверждён — проверяем в спайке (день 0); запасной путь: гнать снапшоты обычным Trystero-action (он unordered-надёжный — чуть хуже, но работает).
- **STUN пачкой** (несколько провайдеров, включая российский): `stun.l.google.com:19302` + `stun.sipnet.ru:3478` + 1-2 других. Google STUN из РФ пока работает, но под риском блокировки — стек серверов ничего не стоит.
- **TURN**: Cloudflare — **нельзя** (IP Cloudflare режутся в РФ до ~16КБ/соединение с июня 2025). Если прямой коннект не пройдёт (шанс 10-20%, главный риск — CGNAT у мобильных/некоторых домашних провайдеров): попробовать Metered Open Relay (20 ГБ/мес бесплатно), а надёжный план Б — coturn на дешёвом российском VPS (~200₽/мес) или «белый» IP у одного из двоих (~150₽/мес у провайдера). Для двоих достаточно, чтобы хоть у одного не было CGNAT.
- **Аварийный сигналинг**: страница ручного обмена SDP (скопировал блок → отправил в Telegram → вставил ответ) — ноль зависимостей, переживает блокировку любых релеев. Паттерн проверен (`cjb/serverless-webrtc` сам живёт на GitHub Pages). Если Nostr-релеи недоступны — смена стратегии Trystero на MQTT/BitTorrent это замена одной строки импорта.

### Неткод — «симметричное доверие + один арбитр»

Для двух доверенных игроков с пингом 5-60 мс серверная авторитарность не нужна (весь этот механизм — защита от читеров). Схема:

- **Своё движение — своя власть.** Каждый симулирует себя локально (код идентичен одиночному режиму), задержка на своё движение — ноль по построению. Никакого client-side prediction и reconciliation.
- **Попадания решает стрелок** (favor the shooter): локальный raycast по интерполированному прокси противника → надёжное событие `damage`. Это тот же результат, что даёт lag compensation Valve, но без кода перемотки. Остаточная нечестность («убили за укрытием») ≈ 100-130 мс — как на официальных серверах CS (`cl_interp` 0.1).
- **Арбитр — создатель комнаты**: фазы раунда, таймер, бомба, счёт, спорные ситуации. Каждые 5 с и на границах раундов шлёт кейфрейм полного состояния матча (без позиций) — ограничивает рассинхрон.
- **Гранаты — власть бросившего**: реплицируется событие спавна (позиция/скорость/таймер), дугу оба считают локально для картинки, урон решает бросивший.
- Числа: сим 60 Гц; отправка 30 Гц; пакет: seq uint16, tick uint32, pos 3×float32, yaw+pitch 2×int16, vel 3×int16, флаги анимации/оружия 1-2 байта ≈ 36-40 Б (~1.2 КБ/с — копейки); буфер интерполяции **100 мс** (3 интервала — переживает потерю 2 пакетов подряд); экстраполяция по скорости макс **250 мс**, дальше — заморозка прокси + плашка «connection…»; heartbeat 1 Гц (он же замер пинга).
- Разрыв: 3 с без пакетов → заморозка + «reconnecting», `pc.restartIce()`, затем полный ре-сигналинг; на возврате арбитр шлёт кейфрейм. 15 с → раунд отменяется. Миграция хоста не нужна — вдвоём без второго игрока матча нет.

### Архитектура кода (чтобы соло-режим и сеть делили код)

```
src/
  core/loop.js        — аккумулятор фиксированного шага, тики
  core/events.js      — шина событий (hit, death, bombPlanted, roundEnd…)
  world/map.js        — загрузка GLB, BVH-коллайдер, данные карты (спавны, зоны)
  world/movement.js   — капсульный контроллер (присед; движение в v1 упрощённое —
                        GoldSrc air-strafe/распрыжка это отдельная физика, осознанно в Фазе 5)
  player/entity.js    — Player, управляется контроллером через общий интерфейс:
    LocalPlayerController   (клавиатура/мышь → локальная симуляция)
    RemoteProxyController   (кольцевой буфер снапшотов → интерполяция)
    BotController           (для тестов/тира — тот же интерфейс)
  combat/weapons.js   — характеристики, hitscan, отдача/разброс, урон
  game/rules.js       — раунды/бомба/экономика: чистые детерминированные функции от событий
  net/transport.js    — интерфейс Transport: send(channel,msg)/onMessage
    WebRTCTransport   (Trystero + ручной unreliable-канал)
    LoopbackTransport (искусственные задержка/джиттер/потери — вся сеть тестируется
                       в двух вкладках на одной машине без интернета)
  ui/…                — HUD, меню покупки, killfeed, scoreboard, радар
```

Вся боевая/раундовая логика подписана на шину событий и не знает о сети; `NetworkAdapter` пересылает локальные события в канал и вливает удалённые обратно в шину. Правила урона/раундов — чистые функции, чтобы оба клиента приходили к одинаковым результатам.

Сборка: **без бандлера** — ES-модули + importmap, three/three-mesh-bvh/trystero **вендорим в репозиторий** (`vendor/`), чтобы не зависеть от доступности CDN из РФ. Деплой — GitHub Pages из main.

## 2. Игровые константы (стартовые, тюним по ощущениям)

- Скорость: бег 250 u/s ≈ **6.35 м/с** (с ножом 260, с AWP 210, с AK 221); присед ×0.34; ходьба (Shift) ×0.52
- Гравитация 800 u/s² ≈ **20.3 м/с²**; прыжок ≈ 6.8 м/с вертикальной скорости
- Капсула: радиус 0.4 м, высота 1.83 м (присед 1.37 м), глаза 1.63 м
- Раунд 1:55, freezetime 3-5 с, бомба **40 с**, дефуз 10 с (5 с с кусачками), победа — **13 раундов** (MR12, настраиваемо), смена сторон после 12
- Деньги: старт $800, потолок $16000, килл +$300 (нож +$1500, AWP +$100), победа +$3250, серия поражений +$1400→$3400 (шаг $500), закладка +$300 команде (+$800 если раунд проигран)
- Оружие (урон в тело / множитель головы ×4, кроме AWP). **Пробитие брони — пооружейное** (плоские «−50%» ломают баланс): урон по бронированной зоне = урон × пробитие.

## 4. Риски и запасные пути

| Риск | Вероятность | План Б |
|---|---|---|
| NAT/CGNAT — прямой P2P не устанавливается | 10-20% | Metered TURN (20ГБ/мес бесплатно, но нужна регистрация и ключ будет виден в публичном JS — для приватной игры ок) → coturn на RU VPS ~200₽/мес → белый IP у одного из двоих |
| Nostr-релеи недоступны из РФ | низкая | Trystero MQTT/BitTorrent (1 строка) → ручной SDP через Telegram |
| github.io недоступен/затроттлен из РФ | низкая-средняя | игра — чистая статика: zip другу + `npx serve` локально, либо зеркало на RU-хостинге |
| Google STUN заблокируют | средняя (тренд 2025-26) | стек STUN: sipnet.ru + другие не-Google уже в конфиге с первого дня |
| GLB-карта «дырявая» для коллизий (нет playerclip) | средняя | дев-редактор точек умеет ставить невидимые коллизионные боксы в map-data.json |
| Свёрнутая вкладка = замершая симуляция (rAF стоит) | точно случится | тикать сим в setInterval при document.hidden; вдвоём оба всё равно в игре |
| Звук не стартует | точно без обработки | AudioContext.resume() по первому клику (экран «нажми чтобы играть») |
| Спорные одновременные события (двойной килл, килл-в-момент-дефуза) | средняя | правила арбитра: его events-канал упорядочен, первый пришедший выигрывает тай-брейк |
| Симметричное доверие «чувствуется нечестно» | низкая при 5-60 мс | зафиксирован полный запасной дизайн: хост-авторитарная сима + prediction у гостя (описан в исследовании, осознанно НЕ делаем пока не понадобится) |

## 5. Тестирование

- **Каждая фаза**: Playwright headless (как в jeep-проекте) — скриншоты ключевых точек карты, ассерты на FPS/ошибки консоли.
